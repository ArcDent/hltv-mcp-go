# HLTV MCP 前端重设计 — 电竞数据看板

## 目标

替换当前的终端绿暗黑风前端，重新设计为**中文优先、电竞主题、全屏铺满**的数据看板。
解决当前页面的 5 个问题：字号过小、全英文不便阅读、字号不一致、输入框对比度低、结果格式化展示不足。

## 设计方向

- **风格**：深蓝灰 + 金色，HLTV 品牌电竞风
- **色板**：背景 `#0d1117` / 面板 `#161b22` / 主色(金) `#f0c040` / 警示橙 `#ff8c42`
- **字体**：系统中文优先（PingFang SC / Microsoft YaHei），数值用等宽数字
- **布局**：左侧导航 200px + 主内容区全屏铺满，padding 32px，卡片间距 12px

## 色彩体系

| 角色 | 色值 | 用途 |
|------|------|------|
| 主背景 | `#0d1117` | 主内容区底色 |
| 面板 | `#161b22` | 卡片、导航栏、输入框底色 |
| 边框 | `#30363d` | 组件边框，hover 时过渡到金色 |
| 主色(金) | `#f0c040` | 比分数字、活跃标签、表头底线、聚焦边框 |
| 警示(橙) | `#ff8c42` | LIVE 标签、比赛中状态指示 |
| 正文 | `#e6edf3` | 队伍名、标题、统计数值 |
| 次要文字 | `#8b949e` | 标签、日期、slug、placeholder |

## 字号阶梯

| 层级 | 大小/字重 | 用途 |
|------|----------|------|
| 比分 | 28px / 700 | 比赛卡片比分数字 |
| 指标值 | 26px / 700 | Dashboard 统计卡片数值 |
| 队伍名 | 16px / 600 | 比赛卡片队名 |
| 导航项 | 15px | 侧栏菜单项 |
| 正文 | 15px | 新闻标题、搜索结果行 |
| 标签 | 13px | Section 标题、日期、slug |
| 辅助 | 12px | placeholder 文字、状态说明 |

主内容区：flex: 1，padding 32px，垂直滚动。

## 6 页面详细设计

### 页面 1 — 总览 (Dashboard)

- 4 个统计卡片横排（运行时间 / Go 版本 / 内存 / 缓存条目），26px 数值
- 系统状态面板：4 行状态指示器，绿点+服务名+详情
- 所有卡片和面板均为 `#161b22` 底色 + `#30363d` 边框

### 页面 2 — 赛程 (Matches)

- 3 个标签切换按钮：今日 / 即将开始 / 近期赛果
- 比赛卡片：队名(16px) + 比分或时间(28px 金色) + 赛事标签(13px 金色边框)
- 进行中的比赛：左侧橙色 LIVE 标签
- 比赛卡片 hover：边框颜色过渡到金色
- 底部筛选输入框：placeholder "输入队伍名筛选..."

### 页面 3 — 队伍 (Teams)

- 大号搜索框(15px字号) + "搜索"按钮(金色)
- 结果列表：编号(金色) + 队名(16px) + ID标签 + slug
- 空状态：提示"输入队名搜索"
- 搜索结果 hover：边框金色过渡

### 页面 4 — 选手 (Players)

- 同队伍页面结构
- 结果列表：编号 + 选手名 + ID标签 + slug

### 页面 5 — 新闻 (News)

- 2 个标签切换：实时新闻 / 归档新闻
- 新闻条目：编号(金色) + 标题(15px) + 日期(13px 灰色)
- hover：标题文字过渡金色、边框金色

### 页面 6 — 缓存 (Cache)

- 3 个统计卡片横排（条目数 / 命中 / 未命中）
- 清除缓存按钮（橙色警示色）+ 刷新按钮（灰色）
- 清除后显示"缓存已清除"反馈文字（金色，2.5s 后消失）

## 交互规范

- 所有导航项、按钮、比赛卡片、搜索结果行 hover 时 `border-color` 从 `#30363d` 过渡到 `rgba(240,192,64,0.3)`
- 搜索框聚焦时 `border-color` 变为 `#f0c040`，placeholder 颜色 `#484f58`（确保可读）
- 列表项添加 stagger 入场动画（`animation-delay: i*30ms`）
- 页面切换不重新挂载组件（React Router 保持状态）

## 数据展示细节

### 队伍名展示格式

两行展示：
- 第一行：英文原名 16px / 600（如 `Spirit`）
- 第二行：中文昵称 12px / 灰色 `#8b949e`（如 `绿龙`）
- 使用 `localization.LookupTeam` 获取 `Display` 和 `Colloquial` 字段
- 若队伍不在 catalog 中，第二行留空

### 比赛状态判定

基于 `scheduled_at` 时间判断：
- `Score` 有值 → 已结束，展示比分（金色 28px）
- `scheduled_at` 在当前时间 ±2 小时内 → 展示"即将开始"橙色标签（`#ff8c42`），替代比分位显示倒计时风格时间
- `scheduled_at` > 2 小时后 → 展示时间（常规金色 28px），无特殊标签

### 侧栏滚动

侧栏 `position: sticky; top: 0; height: 100vh`，主内容区 `overflow-y: auto`。滚动赛程/新闻列表时侧栏始终可见。

## 技术约束

- React 18 + TypeScript + Vite + Tailwind CSS v4
- 产物通过 `//go:embed dist/*` 嵌入 Go 二进制
- 前端 API 调用通过 `/api/*` 代理
- 无需额外 npm 依赖（react-router-dom 已安装）
